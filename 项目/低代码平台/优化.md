## Q5：为什么要区分深拷贝与浅拷贝？

**A**：平衡安全性与性能。

- **深拷贝**：复制组件/保存历史快照/更新嵌套数据。
- **浅拷贝**：更新单一属性/根节点优化渲染。
- 历史管理：`useHistoryManager` 保存快照（深拷贝）。

---

## Q6：性能优化有哪些？

**A**：

- 虚拟滚动 → 大量组件时提升渲染性能。
- 增量更新 → 只重渲染变更节点。
- Schema 缓存 → 复杂 Schema 结果缓存。
- 懒加载组件 → 按需加载业务组件。
- 智能拷贝 → 深浅拷贝按场景使用。

## Q7：运营发布流程？

**A**：函数式工作流。

- 创建项目 → 拖拽表单 → 配置属性 → 预览测试 → 生成 Schema → 发布活动。
- 发布配置：包含活动 ID、Schema、时间、目标人群、ABTest 配置。
- 部署：`deployToProduction`。

```typescript
const operationWorkflow = pipe(
  createProject,
  designForm,
  configProperties,
  previewAndTest,
  generateSchema,
  publishActivity
);
```
